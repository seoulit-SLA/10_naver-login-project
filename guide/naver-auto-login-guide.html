Naver OAuth 자동 로그인
아키텍처 가이드

첫 로그인에서만 Naver OAuth를 수행하고, 이후에는 자체 세션 토큰으로 자동 로그인하는 패턴입니다. Unity · Electron · 모바일 등 데스크톱/게임 클라이언트 프로젝트에 재사용할 수 있습니다.

개요

브라우저 기반 OAuth는 최초 1회만 실행합니다. 서버는 Naver로부터 받은 accessToken과 refreshToken을 DB에 암호화 저장하고, 클라이언트에는 자체 발급 sessionToken만 전달합니다.

첫 로그인

브라우저 → Naver OAuth → 서버 토큰 저장 → sessionToken 발급

이후 로그인

클라이언트 sessionToken → 서버 검증 → OAuth 없이 진입

토큰 만료

refreshToken으로 갱신, 실패 시 OAuth 재실행

✓
적용 대상: Unity, Unreal, Electron, Tauri 등 로컬 HTTP 콜백(127.0.0.1)을 사용하는 데스크톱/게임 클라이언트

설계 원칙

Naver 토큰은 서버 전용
refreshToken을 클라이언트(앱)에 저장하지 않습니다. PlayerPrefs, localStorage 등은 쉽게 노출됩니다.
자동 로그인은 자체 sessionToken
Naver 토큰은 "Naver API 호출 권한"이고, 앱 세션과는 목적이 다릅니다. JWT 등 서버 발급 토큰으로 인증합니다.
OAuth는 예외 상황에서만
최초 로그인, refreshToken 만료/폐기, 사용자 연동 해제, sessionToken 무효화 시에만 브라우저 OAuth를 다시 엽니다.
Graceful Degradation
자동 로그인 실패 시 사용자에게 재로그인 버튼을 제공하고, OAuth 플로우로 자연스럽게 폴백합니다.

토큰 역할 분리

토큰 저장 위치 용도 수명
naver_access_token 서버 DB (암호화) Naver Open API 호출 약 1시간 (Naver 정책)
naver_refresh_token 서버 DB (암호화) accessToken 갱신 장기 (만료/폐기 가능)
sessionToken 클라이언트 로컬 앱 자동 로그인 인증 서버 설정 (예: 30일)
⚠
주의: Naver refreshToken도 영구적이지 않습니다. 사용자가 앱 연동을 해제하거나 Naver 정책 변경 시 재OAuth가 필요합니다.

첫 로그인 흐름

── 첫 로그인 (OAuth) ── Client → GET /auth/naver (브라우저 열기) Browser → Naver 로그인 페이지 Naver → GET /auth/naver/callback?code=... Server → Naver token endpoint (code → accessToken, refreshToken) Server → Naver profile API (uid, email, name) Server → DB upsert (profile + encrypted tokens) Server → JWT sessionToken 발급 Server → Redirect http://127.0.0.1:{port}/callback?token={sessionToken} Client → HttpListener로 token 수신 → 로컬 저장

Unity 콜백 URL 예시

// 클라이언트가 수신 대기하는 prefix
http://127.0.0.1:7777/naver-login/

// OAuth 성공 후 서버가 리다이렉트하는 URL
http://127.0.0.1:7777/naver-login/?token=eyJhbGciOiJIUzI1NiIs...

자동 로그인 흐름

── 앱 시작 / 재실행 ── Client → 로컬 sessionToken 존재 여부 확인 Client → POST /auth/me Authorization: Bearer {sessionToken} Server → JWT 검증 → uid 조회 → 사용자 정보 반환 Client → 로그인 완료, 메인 화면 진입 (OAuth 없음)
✓
이 경로가 일반적인 재로그인입니다. 사용자는 브라우저를 열 필요가 없습니다.

토큰 갱신 흐름

── sessionToken 만료 또는 /auth/me 401 ── Client → POST /auth/refresh Authorization: Bearer {sessionToken} Server → DB에서 naver_refresh_token 조회 (복호화) Server → Naver token refresh API 호출 Server → 새 accessToken/refreshToken DB 저장 Server → 새 sessionToken 발급 → 클라이언트에 반환 Client → sessionToken 교체 저장

재인증 흐름 (OAuth 재실행)

다음 상황에서는 자동 갱신이 불가능하므로 OAuth를 다시 실행합니다.

상황 서버 응답 클라이언트 동작
로컬 sessionToken 없음 — 로그인 버튼 → OAuth
sessionToken 만료 + refresh 실패 401 + code: REAUTH_REQUIRED 로컬 토큰 삭제 → OAuth
Naver refreshToken 폐기 401 + code: NAVER_TOKEN_REVOKED 로컬 토큰 삭제 → OAuth
사용자 로그아웃 200 로컬 토큰 삭제 → 로그인 화면

DB 스키마

CREATE TABLE users (
  id                    INT          NOT NULL AUTO_INCREMENT,
  uid                   VARCHAR(64)  NOT NULL,          -- Naver profile id
  email                 VARCHAR(255) NOT NULL,
  name                  VARCHAR(100) NOT NULL,
  naver_access_token    TEXT,                              -- AES 암호화
  naver_refresh_token   TEXT,                              -- AES 암호화
  token_expires_at      DATETIME,                            -- accessToken 만료 시각
  created_at            TIMESTAMP    NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at            TIMESTAMP    NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (id),
  UNIQUE KEY uq_users_uid (uid)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

JWT sessionToken 페이로드 (예시)

{
  "sub": "naver_uid_12345",     // users.uid
  "iat": 1710000000,
  "exp": 1712592000              // 30일 등 서버 정책
}

API 명세

OAuth

GET /auth/naver — Naver OAuth 시작 (브라우저 리다이렉트)

GET /auth/naver/callback — OAuth 콜백, DB 저장 후 클라이언트 콜백 URL로 리다이렉트

Session

POST /auth/me — sessionToken 검증, 사용자 정보 반환

POST /auth/refresh — sessionToken + Naver refreshToken 갱신

POST /auth/logout — 세션 무효화 (선택: JWT blacklist)

POST /auth/me

// Request
Authorization: Bearer eyJhbGciOiJIUzI1NiIs...

// Response 200
{
  "uid": "12345678",
  "email": "user@example.com",
  "name": "홍길동"
}

// Response 401
{
  "message": "세션이 만료되었습니다.",
  "code": "SESSION_EXPIRED"
}

OAuth 콜백 → 클라이언트 리다이렉트

// 성공
res.redirect(`http://127.0.0.1:7777/naver-login/?token=${sessionToken}`);

// 실패
res.redirect(`http://127.0.0.1:7777/naver-login/?error=auth_failed`);

OAuth 콜백 핵심 로직

passport.use(new NaverStrategy(config, async (accessToken, refreshToken, profile, done) => {
  const user = {
    uid: profile.id,
    email: profile.email,
    name: profile.name || profile.nickname,
  };

  // 1. DB upsert (tokens encrypted)
  await upsertUser(user, accessToken, refreshToken);

  // 2. Issue session JWT
  const sessionToken = signSessionToken(user.uid);

  return done(null, { ...user, sessionToken });
}));

클라이언트 구현 가이드

앱 시작 시 의사결정 트리

OnAppStart()
│
├─ sessionToken = LoadFromLocal()
│
├─ sessionToken == null
│   └─ ShowLoginButton()  → 클릭 시 OAuth
│
└─ sessionToken != null
    └─ response = POST /auth/me
        ├─ 200 → EnterMainScreen()
        ├─ 401 SESSION_EXPIRED → TryRefresh()
        │   ├─ success → SaveToken → EnterMainScreen()
        │   └─ fail → ClearToken → ShowLoginButton()
        └─ 401 REAUTH_REQUIRED → ClearToken → ShowLoginButton()

Unity HttpListener 콜백 처리

private void HandleCallback(HttpListenerRequest request)
{
    var query = HttpUtility.ParseQueryString(request.Url.Query);
    var token = query["token"];
    var error = query["error"];

    if (!string.IsNullOrEmpty(token))
    {
        PlayerPrefs.SetString("sessionToken", token);
        PlayerPrefs.Save();
        EnterMainScreen();
    }
    else
    {
        Debug.LogError($"Login failed: {error}");
    }
}
ℹ
프로덕션에서는 PlayerPrefs 대신 OS Keychain(Windows Credential Manager, macOS Keychain) 또는 암호화된 로컬 저장소 사용을 권장합니다.

보안 가이드

항목 권장 비권장
Naver refreshToken 서버 DB AES 암호화 클라이언트 저장, 평문 DB
sessionToken JWT (HS256/RS256), 짧은 TTL + refresh uid만 base64 인코딩
통신 HTTPS (프로덕션) HTTP (로컬 개발만)
콜백 URL 127.0.0.1 고정 포트 임의 포트, 외부 URL
로그아웃 클라이언트 토큰 삭제 + 서버 blacklist 클라이언트만 삭제
✕
절대 금지: .env의 Naver Client Secret, DB refreshToken, JWT Secret을 Git에 커밋하거나 클라이언트 번들에 포함하지 마세요.

구현 체크리스트

Phase 1 — DB & Server Foundation
  • users 테이블 스키마 마이그레이션 (uid, token 컬럼)
  • 토큰 AES 암호화/복호화 유틸 (services/crypto.js)
  • JWT 발급/검증 유틸 (services/session.js)
  • OAuth 콜백: DB 저장 + sessionToken 발급 + 클라이언트 리다이렉트
Phase 2 — Auth API
  • POST /auth/me — JWT 검증, 사용자 반환
  • POST /auth/refresh — Naver refresh + sessionToken 재발급
  • POST /auth/logout — 세션 무효화
Phase 3 — Client
  • HttpListener 콜백에서 sessionToken 파싱·저장
  • 앱 시작 시 /auth/me 자동 로그인
  • sessionToken 없을 때만 OAuth 브라우저 열기
  • 401 응답 시 refresh → 실패 시 OAuth 폴백
Phase 4 — Test
  • 첫 로그인: OAuth → token 저장 → 메인 진입
  • 재실행: OAuth 없이 자동 로그인
  • sessionToken 만료: refresh 성공
  • refreshToken 폐기: OAuth 재실행
  • 로그아웃: 토큰 삭제 후 로그인 화면

에러 코드 참조

code HTTP 의미 클라이언트 처리
SESSION_EXPIRED 401 sessionToken 만료 /auth/refresh 시도
SESSION_INVALID 401 JWT 서명 불일치 토큰 삭제 → OAuth
NAVER_TOKEN_REVOKED 401 Naver refreshToken 무효 토큰 삭제 → OAuth
REAUTH_REQUIRED 401 재인증 필요 토큰 삭제 → OAuth
USER_NOT_FOUND 404 DB에 사용자 없음 토큰 삭제 → OAuth
