이 문서는 초보자가 직접 모든 코드를 외워서 작성하는 방식이 아니라, 코딩 에이전트에게 단계별 프롬프트를 주면서 Express 서버, MySQL DB, 관리자 로그인, bcrypt 비밀번호 암호화, JWT 쿠키 인증, PM2 실행까지 완성하는 교재입니다.

사용 규칙

아래 단계는 위에서 아래로 한 번에 하나씩 진행합니다. 각 단계의 프롬프트를 에이전트에게 그대로 붙여넣고, 에이전트가 작업을 마치면 설명을 읽고 다음 단계로 넘어갑니다.

DB 연결 정보와 테이블 설계

관리자 계정을 저장할 MySQL DB와 admin_user 테이블을 준비합니다.

에이전트 프롬프트DB 설계
MySQL DB를 사용하는 관리자 로그인 서버를 만들 수 있도록 DB 설계를 추가해줘.

요구사항:
- database 이름은 gamedb
- 관리자 테이블 이름은 admin_user
- 컬럼은 id, username, password, created_at
- id는 AUTO_INCREMENT PRIMARY KEY
- username은 VARCHAR(50), UNIQUE
- password는 VARCHAR(255)
- created_at은 생성 시각 자동 저장
- index.js에서 mysql2/promise로 DB pool 생성
- DB_HOST, DB_PORT, DB_USER, DB_PASSWORD, DB_NAME 환경 변수를 지원
- 환경 변수가 없으면 localhost, 3306, root, 1234, gamedb 기본값 사용
- 서버 시작 시 admin_user 테이블이 없으면 자동 생성되게 해줘
설명:

admin_user는 관리자 목록을 저장하는 표입니다. 비밀번호 컬럼은 원본 비밀번호가 아니라 해시 문자열을 저장합니다.

초기 관리자 계정 자동 생성

처음 서버를 켰을 때 admin / 1234 계정을 자동으로 만듭니다.

에이전트 프롬프트초기 계정
서버 시작 시 초기 관리자 계정을 자동으로 생성하는 기능을 추가해줘.

요구사항:
- admin_user 테이블에 username이 admin인 계정이 있는지 먼저 확인
- 없으면 username은 admin, password는 1234로 등록
- 단, 1234를 그대로 저장하지 말고 bcryptjs로 암호화해서 저장
- bcrypt salt rounds는 10 사용
- 이미 admin 계정이 있으면 중복 생성하지 않기
- 작업 후 DB에 어떤 값이 저장되는지 초보자용으로 설명해줘
핵심:

로그인할 때는 사용자가 1234를 입력하지만, DB에는 1234가 보이면 안 됩니다.

서버 라우트 연결

브라우저 주소와 HTML 파일, 로그인 처리 주소를 연결합니다.

로그인 비교와 JWT 발급

서버가 아이디와 비밀번호를 받아 DB와 비교한 뒤 JWT를 발급합니다.

에이전트 프롬프트로그인 처리
POST /admin/login 로그인 처리 코드를 완성해줘.

흐름:
1. 클라이언트에서 username, password 받기
2. admin_user 테이블에서 username으로 관리자 조회
3. 관리자가 없으면 로그인 실패 응답
4. 관리자가 있으면 bcrypt.compare(password, user.password)로 비교
5. 비밀번호가 틀리면 로그인 실패 응답
6. 맞으면 JWT 발급
7. JWT payload에는 id, username, role: "admin" 포함
8. JWT_SECRET은 환경 변수 사용, 없으면 개발용 기본값 사용
9. 토큰 만료 시간은 1시간
10. 발급한 JWT를 auth_token 쿠키에 저장
11. 쿠키 옵션은 httpOnly: true, sameSite: "lax", maxAge: 1시간
12. 성공하면 /admin으로 redirect

초보자가 이해할 수 있게 각 단계에 주석을 달아줘.
설명:

JWT는 로그인 성공 증명서입니다. 쿠키는 그 증명서를 브라우저에 보관하는 저장소입니다.

8

인증 미들웨어 만들기

관리자 페이지 요청마다 쿠키의 JWT를 검사합니다.

에이전트 프롬프트인증 보호
관리자 페이지 보호용 인증 미들웨어를 만들어줘.

함수 이름:
- requireAdminLogin

동작:
1. req.cookies.auth_token에서 JWT 읽기
2. 토큰이 없으면 /admin/login으로 redirect
3. 토큰이 있으면 jwt.verify로 검증
4. 검증 성공 시 req.user에 payload 저장 후 next()
5. 검증 실패 시 쿠키 삭제 후 /admin/login으로 redirect

적용:
- GET /admin 라우트에 requireAdminLogin 적용

초보자가 "미들웨어가 무엇인지" 이해할 수 있도록 설명도 함께 작성해줘.

관리자 CRUD API 만들기

관리자 계정을 조회, 등록, 수정, 삭제하는 API를 만듭니다.

에이전트 프롬프트CRUD
admin_user 테이블 CRUD API를 추가해줘.

모든 API는 requireAdminLogin 미들웨어로 보호해줘.

API:
- GET /api/admin-users
  - 관리자 목록 조회
  - id, username, created_at만 응답
  - password는 절대 응답하지 않기

- POST /api/admin-users
  - username, password를 받아 새 관리자 등록
  - password는 bcrypt로 암호화 저장

- PUT /api/admin-users/:username/password
  - 특정 관리자의 비밀번호 변경
  - 새 비밀번호도 bcrypt로 암호화 저장

- DELETE /api/admin-users/:username
  - 특정 관리자 삭제

요구사항:
- try/catch로 DB 오류 처리
- 성공과 실패를 JSON으로 응답
- SQL injection 방지를 위해 ? placeholder 사용
- 초보자용으로 CRUD 뜻도 설명해줘.

순서 위치 동작 초보자 설명

문제 해결 프롬프트

막혔을 때는 아래 프롬프트를 에이전트에게 주면 됩니다.

DB 연결 실패MySQL
MySQL 연결이 실패했어. 현재 프로젝트의 index.js DB 설정과 내 MySQL 상태를 확인해서 원인을 찾아줘.

확인할 것:
- MySQL 서버 실행 여부
- DB_HOST, DB_PORT, DB_USER, DB_PASSWORD, DB_NAME 값
- gamedb 데이터베이스 존재 여부
- admin_user 테이블 존재 여부
- 에러 메시지 원인 설명
- 필요한 수정 직접 적용
로그인 실패bcrypt/JWT
admin / 1234 로그인이 실패해. 로그인 흐름 전체를 점검해줘.

확인할 것:
- admin_user 테이블에 admin 계정 존재 여부
- password 컬럼에 bcrypt 해시가 저장되어 있는지
- bcrypt.compare 코드가 맞는지
- POST /admin/login에서 username, password를 제대로 받는지
- JWT 발급 전에 에러가 나는지
- 문제를 찾으면 직접 수정
PM2 이상Windows
PM2 상태가 이상해. Windows PowerShell 기준으로 pm2.cmd를 사용해서 점검해줘.

확인할 것:
- pm2.cmd list
- pm2.cmd logs
- 멈춘 PM2 관련 node 프로세스 여부
- ecosystem.config.js 설정
- 앱이 online인지
- 필요하면 PM2 프로세스를 정리하고 다시 start/save
