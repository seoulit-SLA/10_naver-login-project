게임 씬 흐름 &
최고 점수 서버 연동 가이드

로그인 후 홈 → 게임 → 게임오버 → 홈으로 이어지는 씬 구조, 서버 통신 중 로딩 UI, 최고 점수 조회·제출 API를 처음부터 끝까지 따라 만들 수 있는 문서입니다. 선행 작업: Naver OAuth 자동 로그인 가이드

개요

이번에 추가된 기능은 크게 씬 흐름, 로딩 UI, 최고 점수 서버 연동 세 가지입니다. 각 씬에는 XXXMain 컴포넌트가 붙은 GameObject가 있고, UI는 Inspector의 [SerializeField]로 연결합니다.

씬 흐름

App → Title → Home → Game → GameOver → Home

로딩 UI

서버 통신 중 GlobalLoadingCanvas 표시

최고 점수

Home 진입 시 조회, Game 종료 시 신기록만 제출

싱글톤

AppMain, AuthClient, GlobalLoadingCanvas (DontDestroyOnLoad)

HomeScene — 최고 점수 로드

GameOverScene → HomeScene

게임 규칙

점수는 씬 간 공유를 위해 정적 클래스 GameSession에 저장합니다.

// client/Assets/GameSession.cs
public static class GameSession
{
    public const float GameDurationSeconds = 10f;
    public const int PointsPerClick = 10;

    public static int Score { get; private set; }

    public static void ResetScore() => Score = 0;
    public static void AddScore(int points) => Score += points;
}
  • Home에서 게임 시작 시 GameSession.ResetScore() 호출
  • GameScene에서 클릭 버튼마다 AddScore(10)
  • GameOverScene에서 GameSession.Score를 화면에 표시

서버 DB 마이그레이션

1테이블 생성

cd server
npm run migrate:high-scores

생성되는 테이블: user_high_scores

-- server/migrations/002_user_high_scores.sql
CREATE TABLE IF NOT EXISTS user_high_scores (
  id INT NOT NULL AUTO_INCREMENT,
  uid VARCHAR(64) NOT NULL,
  best_score INT NOT NULL DEFAULT 0,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (id),
  UNIQUE KEY uq_user_high_scores_uid (uid),
  KEY idx_user_high_scores_best_score (best_score DESC)
);
✓
uid는 users 테이블의 네이버 uid와 동일합니다. 한 사용자당 최고 점수 1행(UNIQUE)만 저장됩니다.

서버 API

두 API 모두 POST이며, Authorization: Bearer {sessionToken} 헤더가 필요합니다.

POST /scores/me — 내 최고 점수 조회

항목값
MethodPOST
Body{} (빈 JSON)
성공 200{ "uid", "bestScore", "updatedAt" }
기록 없음bestScore: 0, updatedAt: null
 
// 응답 예시
{
  "uid": "abc123",
  "bestScore": 120,
  "updatedAt": "2026-07-06T07:00:00.000Z"
}

POST /scores/submit — 최고 점수 제출

항목값
MethodPOST
Body{ "score": 150 } — 0 이상 정수
신기록DB 업데이트, isNewRecord: true
기존 이상DB 변경 없음, isNewRecord: false
잘못된 score400 INVALID_SCORE
 
// 신기록 응답 예시
{
  "uid": "abc123",
  "submittedScore": 150,
  "bestScore": 150,
  "previousBestScore": 120,
  "isNewRecord": true,
  "updatedAt": "2026-07-06T07:15:00.000Z"
}
ℹ
서버(highScores.js)와 클라이언트(GameMain) 모두 “기존 최고 점수보다 높을 때만” 갱신합니다. 이중 방어 구조입니다.

Home — 최고 점수 조회

HomeMain.Start()에서 코루틴 LoadBestScore()를 실행합니다.

GlobalLoadingCanvas.Show()
SessionTokenStore.GetToken()
토큰 없으면 scoreText = 0
AuthClient.GetMyBestScore()
성공 시 scoreText 갱신
finally → GlobalLoadingCanvas.Hide()
// client/Assets/HomeMain.cs (핵심)
private IEnumerator LoadBestScore()
{
    GlobalLoadingCanvas.Instance?.Show();
    try
    {
        var sessionToken = SessionTokenStore.GetToken();
        yield return AuthClient.Instance.GetMyBestScore(
            sessionToken,
            result => UpdateScoreText(result.bestScore),
            code => UpdateScoreText(0)
        );
    }
    finally
    {
        GlobalLoadingCanvas.Instance?.Hide();
    }
}

게임 시작 버튼 클릭 시:

GameSession.ResetScore();
AppMain.Instance.GoToGame();

Game — 점수 제출

타이머가 0이 되면 EndGame() → SubmitScoreIfRecordAndGoToGameOver() 코루틴이 실행됩니다.

isGameOver = true, clickButton 비활성화
GlobalLoadingCanvas.Show()
POST /scores/me → bestScore 확인
currentScore > bestScore 이면 POST /scores/submit
아니면 로그만 남기고 스킵
finally → Hide() + GoToGameOver()
API 실패해도 GameOverScene으로 이동
// client/Assets/GameMain.cs (핵심 로직)
if (currentScore <= bestScoreResponse.bestScore)
{
    // 제출 스킵
    yield break;
}

yield return AuthClient.Instance.SubmitBestScore(
    sessionToken,
    currentScore,
    result => { /* 로그: isNewRecord */ },
    code => { /* 경고 로그 */ }
);

GameOverScene에서는 서버 통신 없이 GameSession.Score(이번 판 점수)만 표시합니다.

AuthClient

AuthClient는 AppMain이 자동 생성하며 DontDestroyOnLoad로 유지됩니다.

메서드엔드포인트용도
GetMePOST /auth/me자동 로그인
RefreshPOST /auth/refresh토큰 갱신
GetMyBestScorePOST /scores/me최고 점수 조회
SubmitBestScorePOST /scores/submit최고 점수 제출

응답 DTO

public class ScoreMeResponse
{
    public string uid;
    public int bestScore;
    public string updatedAt;
}

public class ScoreSubmitResponse
{
    public string uid;
    public int submittedScore;
    public int bestScore;
    public int previousBestScore;
    public bool isNewRecord;
    public string updatedAt;
}

통합 테스트

아래 순서대로 Play Mode에서 확인하세요. 체크박스는 브라우저에서 직접 체크할 수 있습니다.

  • 서버 실행 + npm run migrate:high-scores 완료
  • 네이버 로그인 → HomeScene 진입
  • Home 진입 시 로딩 표시 → scoreText에 최고 점수(처음이면 0)
  • 게임 시작 → 10초 동안 클릭 → 점수 증가 확인
  • 타이머 종료 → 로딩 표시 → GameOverScene 이동
  • GameOver scoreText = 이번 판 점수
  • 홈 버튼 → Home scoreText = 갱신된 최고 점수
  • 같은 점수 이하로 다시 플레이 → Home 최고 점수 변화 없음
  • 더 높은 점수로 플레이 → Home 최고 점수 갱신
  • Console 로그: [HOME], [GAME], [SCORE] 태그 확인

로그 확인 예시

[HOME] best score loaded | bestScore=120
[GAME] score submitted | current=150 best=150 isNewRecord=True
[GAME] score not submitted | current=80 best=150

문제 해결

증상원인 / 해결
scoreText가 항상 0 sessionToken 없음 → 재로그인. AuthClient null → App 씬부터 Play
로딩 UI가 안 보임 AppMain Inspector에 GlobalLoadingCanvas Prefab 미연결
점수가 서버에 안 올라감 이번 점수 ≤ 최고 점수(정상). Console score not submitted 확인
401 Unauthorized sessionToken 만료 → 재로그인 또는 /auth/refresh
500 서버 오류 user_high_scores 테이블 미생성 → migrate 실행
씬 전환 안 됨 Build Settings에 5개 씬 등록 확인
AppMain.Instance is null App 씬이 아닌 개별 씬만 Play → App 씬부터 실행
