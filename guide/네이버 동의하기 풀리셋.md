# 개발 노트

## 서버 명령어

### 서버 상태 확인
```powershell
# 포트 3000 사용 중인 프로세스 확인
netstat -ano | findstr ":3000"

# Node.js 프로세스 목록
Get-Process -Name "node" | Select-Object Id, ProcessName, StartTime
```

### 서버 종료
```powershell
# PID로 종료 (포트 3000 확인 후 해당 PID 사용)
Stop-Process -Id <PID>

# node 프로세스 전체 종료 (주의: 모든 node 프로세스 종료됨)
Get-Process -Name node | Stop-Process
```

### 서버 시작
```powershell
# server 폴더에서 실행
node app.js
```

## 구현 내용

### NaverAuthDebugWindow (Unity 에디터 전용)
- 경로: `client/Assets/Editor/NaverAuthDebugWindow.cs`
- 메뉴: `Tools > Naver Auth 초기화`

| 버튼 | 동작 |
|------|------|
| 로컬 토큰만 삭제 | PlayerPrefs에서 세션 토큰 삭제 |
| 완전 초기화 | `POST /debug/reset` 호출 → DB 유저 행 삭제 + Naver 토큰 폐기 + 로컬 토큰 삭제 |

### /debug/reset 엔드포인트 (서버)
- 역할: 테스트용 유저 완전 초기화
- Body: `{ sessionToken }`
- 동작: DB `users` 행 DELETE + Naver `grant_type=delete` 토큰 폐기
- **운영 환경 배포 금지**

## Naver OAuth 동의 화면 관련

### 동의 화면(동의하기)이 뜨는 조건
- 해당 네이버 계정으로 해당 앱을 **처음** 인증할 때
- `grant_type=delete`로 Naver access_token을 **폐기한 후** 재로그인할 때

### 동의 화면이 안 뜨는 조건
- 이미 로그인한 적 있고 토큰이 유효한 상태에서 재로그인할 때

### 테스트 시 동의 화면 재표시 방법
에디터 창(`Tools > Naver Auth 초기화`) → **완전 초기화** 버튼

내부 동작:
1. `POST /debug/reset` → DB 유저 행 DELETE + Naver `grant_type=delete` 토큰 폐기
2. 로컬 PlayerPrefs 토큰 삭제
3. 이후 로그인 시 동의 화면 재표시됨

> **주의:** 서버가 새 코드로 실행 중이어야 `/debug/reset`이 동작함.
> 서버 재시작 없이 코드만 수정하면 404가 반환되고 토큰 폐기가 안 됨 → 동의 화면 미표시.

## .env 주의사항

| 항목 | 상태 |
|------|------|
| `DB_NAME` | 비어 있음 (app.js는 `gamedb` 하드코딩으로 동작) |
| `JWT_SECRET` | 기본값 그대로 → 운영 전 반드시 변경 필요 |
| `ENCRYPTION_KEY` | 32바이트 hex (정상) |
