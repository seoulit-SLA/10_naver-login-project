Unity-Server Error Protocol

Unity 클라이언트와 Node.js 서버 사이에서 공통으로 사용하는 통신/에러 규약입니다. Linear Dark 스타일의 단일 문서로 유지합니다.

1. Common Response

정상 응답은 각 API 목적에 맞는 JSON을 반환하고, 에러 응답은 아래 공통 포맷을 사용합니다.

{
  "code": "SERVER_ERROR",
  "message": "서버 내부 오류가 발생했습니다."
}

2. Client Timeout Rule

  • 모든 Unity API 요청은 5초 timeout을 사용합니다.
  • 5초 안에 응답이 없으면 REQUEST_TIMEOUT으로 처리합니다.
  • 서버가 꺼져 있거나 연결 실패 시 NETWORK_ERROR로 처리합니다.
  • 클라이언트는 모든 실패를 UISystemMessage 팝업으로 표시합니다.

3. API Paths

Path Method Purpose
/healthGET서버 생존 확인
/auth/mePOST세션 토큰 검증 및 사용자 조회
/auth/refreshPOST세션 갱신
/auth/logoutPOST로그아웃
/scores/mePOST내 최고 점수 조회
/scores/submitPOST최고 점수 제출
/scores/rankingsGET랭킹 목록 조회

4. Error Codes

Code HTTP Description Client Action
REQUEST_TIMEOUT-5초 타임아웃시스템 메시지 표시
NETWORK_ERROR-서버 연결 실패시스템 메시지 표시
PARSE_ERROR-응답 JSON 파싱 실패시스템 메시지 표시
AUTH_CONFIG_MISSING500OAuth 환경변수 누락시스템 메시지 표시
SESSION_INVALID401세션 토큰 없음/무효재로그인 유도
SESSION_EXPIRED401세션 만료refresh 시도
USER_NOT_FOUND404사용자 없음세션 삭제 후 로그인 화면
REAUTH_REQUIRED401재로그인 필요세션 삭제 후 로그인 화면
NAVER_TOKEN_REVOKED401네이버 토큰 폐기재로그인 유도
INVALID_SCORE400잘못된 점수 값시스템 메시지 표시
SERVER_ERROR500서버 런타임 에러시스템 메시지 표시

5. Example

{
  "code": "SESSION_EXPIRED",
  "message": "세션이 만료되었습니다."
}

Unity 모든 요청 실패 시 UISystemMessage가 뜨고, dim 클릭 시 닫힙니다.

Server 라우트 예외는 전역 핸들러에서 SERVER_ERROR로 정규화합니다.
